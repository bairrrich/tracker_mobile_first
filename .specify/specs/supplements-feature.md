# Supplements Feature Specification

## Overview

Приложение для учёта витаминов, БАДов и добавок с отслеживанием приёма, запасов и эффективности.

## User Stories

### US-1: Управление добавками (Supplements)
- **Как** пользователь
- **Хочу** создавать и редактировать добавки
- **Чтобы** вести учёт витаминов и БАДов

**Acceptance Criteria:**
- CRUD для добавок
- Поля: название, тип (витамин/минерал/БАД/протеин), форма (таблетки/капсулы/порошок/жидкость), дозировка, единица измерения
- Бренд/производитель (опционально)
- Фото упаковки (опционально)
- Категория (здоровье/спорт/красота/иммунитет)

### US-2: Управление запасами (Inventory)
- **Как** пользователь
- **Хочу** отслеживать количество добавок
- **Чтобы** вовремя пополнять запасы

**Acceptance Criteria:**
- Текущее количество (штук/грамм/мл)
- Минимальный порог для уведомления
- Дата покупки
- Срок годности
- Уведомления о заканчивающихся добавках
- Уведомления о просроченных добавках

### US-3: Приём добавок (Tracking)
- **Как** пользователь
- **Хочу** отмечать приём добавок
- **Чтобы** следить за регулярностью

**Acceptance Criteria:**
- Отметка о приёме (выполнено/пропущено)
- Время приёма
- Дозировка (если отличается от стандартной)
- Заметки о самочувствии (опционально)
- История приёма

### US-4: Расписание (Schedule)
- **Как** пользователь
- **Хочу** настроить расписание приёма
- **Чтобы** не забывать о добавках

**Acceptance Criteria:**
- Частота: ежедневно/еженедельно/по дням недели
- Время приёма (утро/день/вечер/перед тренировкой)
- Дни приёма (понедельник-воскресенье)
- Привязка к еде (до/после/во время)
- Несколько приёмов в день

### US-5: Эффекты и результаты (Effects)
- **Как** пользователь
- **Хочу** отслеживать эффективность
- **Чтобы** понимать какие добавки работают

**Acceptance Criteria:**
- Оценка эффективности (1-5 звёзд)
- Заметки о результатах
- Дата начала приёма
- Дата окончания приёма
- Побочные эффекты (опционально)

### US-6: Отчёты и статистика
- **Как** пользователь
- **Хочу** видеть отчёты о приёме
- **Чтобы** анализировать регулярность

**Acceptance Criteria:**
- Процент выполненных приёмов за период
- Пропущенные приёмы
- Расходы на добавки за период
- Самые эффективные добавки
- Календарь приёма

## Technical Requirements

### Database Schema

**Tables:**
- `supplements` — основные данные о добавках
- `supplement_inventory` — запасы и сроки годности
- `supplement_schedules` — расписание приёма
- `supplement_logs` — история приёма
- `supplement_categories` — категории добавок
- `supplement_brands` — бренды/производители

### Architecture

- **Offline-first**: все данные хранятся локально в IndexedDB
- **Sync**: синхронизация с сервером через /api/sync
- **Repository Pattern**: доступ к данным через репозитории
- **Zustand**: управление состоянием
- **Zod**: валидация форм
- **next-intl**: локализация (RU + EN)

### UI Components

- Формы с валидацией и уведомлениями (тосты)
- Адаптивный дизайн (mobile-first)
- Тема: light/dark/amoled через CSS переменные
- Календарь для выбора дат
- Time picker для выбора времени
