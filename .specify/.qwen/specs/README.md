---

## Specification Lifecycle

1. **Draft** — Initial specification created
2. **In Review** — Under review for approval
3. **Approved** — Ready for implementation
4. **In Progress** — Currently being implemented
5. **Completed** — Implementation finished
6. **Rejected** — Will not be implemented

---

## Creating a New Specification

1. Copy the template from `templates/feature-spec-template.md`
2. Fill in all sections
3. Save as `specs/[feature-name].md`
4. Update the DEVELOPMENT_PLAN.md with status
